주어진 작업 설명을 바탕으로 `.claude/tasks/{slug}.md` 태스크 파일을 생성한다.

## 절차

1. 작업 설명에서 관련 파일을 **직접 읽어서** 현재 코드 상태를 파악한다 — 추측 금지
2. 변경이 필요한 함수명과 라인번호를 정확히 확인한다
3. 아래 형식으로 태스크 파일을 작성한다
4. 파일명은 작업 내용을 나타내는 kebab-case로 한다 (예: `mypage-profile-mobile-layout.md`)

## 태스크 파일 형식

```markdown
# 태스크: {작업 제목}

## 목적
왜 이 작업을 하는지 — 사용자 요청의 의도와 배경

---

## 변경 전 / 변경 후
변경이 눈에 보이는 경우 표나 목록으로 정리

---

## 대상 파일
`경로/파일명.tsx` — 관련 함수명 (line XXX–XXX)

---

## 구체적인 변경 사항

### 1. {변경 항목 제목}
- 무엇을 제거/추가/교체하는지
- 제거 대상 코드 (라인번호 명시)
- 교체할 코드 예시

---

## 주의 사항
- 건드리지 말아야 할 부분
- 사이드 이펙트 가능성

---

## 검증
작업 완료 후 수행할 체크리스트:
1. `pnpm build` 통과
2. `pnpm lint` 통과
3. 브라우저 확인 항목 (화면 크기별)
```

## 주의

- 파일을 읽지 않고 태스크 파일을 작성하지 않는다
- 라인번호는 실제 읽은 결과 기준으로 명시한다
- 데스크탑/모바일 중 변경 범위가 어디까지인지 명확히 구분한다
