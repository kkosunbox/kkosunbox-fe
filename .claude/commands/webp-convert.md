지정한 범위(파일, 디렉터리, 또는 위젯/컴포넌트 이름)의 png/jpg/jpeg 이미지를 webp로 변환하고, 코드의 모든 참조 경로를 함께 수정한다. 화면에 보이는 이미지는 동일하게 유지하면서 용량만 최적화하는 것이 목적이다.

인자: `$ARGUMENTS` — 파일 경로, 디렉터리, 또는 컴포넌트/위젯 이름 (예: `public/images/modal`, `entities/package/assets`, `SupportHero`)

## 절차

### 1. 범위 확정

- 인자가 파일/디렉터리 경로면 그대로 사용한다.
- 인자가 컴포넌트/위젯 이름이면 해당 파일을 찾아 그 안에서 참조하는 png/jpg 파일들을 대상 목록으로 만든다.
- 인자가 없으면 변환하지 않고, 저장소 전체의 png/jpg 후보 목록(경로 + 용량)을 보여준 뒤 범위를 물어본다.
- **제외 대상 (변환 금지):** `public/og-image.png` 등 OG 이미지, favicon 류. 일부 SNS 크롤러가 webp를 지원하지 않는다. 범위에 포함돼 있으면 제외했다고 보고한다.

### 2. 변환

```bash
node scripts/convert-to-webp.mjs <대상...> [--quality=N] [--lossless]
```

- 기본 품질 82. 아이콘·일러스트 등 색 경계가 또렷한 png는 `--lossless` 사용을 고려한다.
- `--delete`는 이 단계에서 쓰지 않는다 (참조 수정·검증 전 원본 삭제 금지).
- 스크립트가 "건너뜀"(webp가 더 큼)으로 표시한 파일은 원본을 유지하고 참조도 수정하지 않는다.

### 3. 참조 경로 수정

변환된 각 파일의 베이스네임으로 저장소 전체를 Grep하여 모든 참조를 `.webp`로 수정한다. 이 프로젝트의 참조 패턴:

- **정적 import** — `import foo from "../assets/foo.png"` (entities/widgets/shared의 assets)
- **public 문자열 경로** — `src="/images/foo.png"`, 메타데이터의 `"/og-image.png"` 등
- CSS `url(...)`, 테스트 코드, `.md` 문서 내 경로도 확인한다.

### 4. 검증

1. `npx tsc --noEmit` — 타입 오류 없음 (정적 import 경로 깨짐 감지)
2. `pnpm lint` — 새 오류 없음
3. 변환 대상의 옛 확장자 참조가 코드에 남아있지 않은지 Grep으로 재확인

### 5. 원본 삭제

검증 통과 후 변환된 파일들의 원본 png/jpg를 삭제한다 (건너뜀 파일 제외).

### 6. 보고

- 변환 파일 수, 총 용량 변화 (스크립트 출력 인용)
- 수정한 참조 파일 목록
- 제외/건너뜀 파일과 사유
